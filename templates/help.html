快速开始

1. 启动系统

运行启动脚本启动Web服务：

# Windows
run.bat

# Linux/macOS
chmod +x run.sh
./run.sh

# 或手动启动
pip install -r requirements.txt
python app.py

2. 访问Web界面

启动后，在浏览器中访问：

http://localhost:5000

如果需要从其他机器访问，请使用服务器的IP地址：

http://YOUR_SERVER_IP:5000

3. 启动监听器

在首页或"主机管理"页面输入端口号，点击"启动监听"按钮。

4. 生成载荷

在首页"载荷生成器"中：

  • 填写您的IP地址（目标机器将连接到此IP）
  • 填写监听端口
  • 选择载荷类型
  • 点击"生成载荷"并复制

5. 执行载荷

在目标机器上执行生成的载荷命令。连接成功后，将在"主机管理"页面看到连接信息。

6. 交互操作

点击"连接"进入Web终端，即可与目标Shell进行交互。

提示：确保防火墙已开放监听端口，否则目标机器无法连接。

监听器管理

什么是监听器？

监听器是一个在指定端口等待连接的服务。当目标机器执行反弹Shell载荷后，会主动连接到监听器，从而建立Shell会话。

创建监听器

  1. 进入"主机管理"页面
  2. 在输入框中输入端口号（1-65535）
  3. 点击"启动监听"按钮
  4. 等待状态变为"运行中"

批量监听

在"快速连接"页面，您可以：

  • 点击预设端口快速启动
  • 输入多个端口（每行一个）批量启动
  • 使用范围格式（如 4444-4449）批量启动

停止监听器

在监听器列表中点击"停止"按钮即可停止监听。停止后所有已建立的连接也会断开。

注意：停止监听器会断开所有连接到该端口的Shell会话。

监听器状态

状态 说明
运行中 监听器正常运行，等待连接
已停止 监听器已停止

载荷生成

什么是载荷？

载荷（Payload）是在目标机器上执行的代码，用于建立与监听器的反向连接。

支持的载荷类型

类型 平台 说明
Bash -i Linux/Unix/macOS 最常用的反弹Shell，依赖/dev/tcp
Python 跨平台 需要Python环境，功能完整
PHP Web服务器 适用于PHP环境
Perl Linux/Unix 需要Perl环境
Ruby 跨平台 需要Ruby环境
Netcat Linux/Unix 简洁高效，需要-e参数支持
Netcat FIFO Linux/Unix 无需-e参数，使用命名管道
PowerShell Windows Windows系统首选
Java 跨平台 适用于Java环境
Lua 跨平台 需要Lua环境
Node.js 跨平台 需要Node.js环境

载荷编码

支持多种编码方式绕过简单过滤：

  • Base64 - 适用于PowerShell等场景
  • URL编码 - 适用于Web场景
  • Hex编码 - 十六进制编码

使用建议

Linux: 优先使用 Bash 或 Python，成功率最高。
Windows: 使用 PowerShell，兼容性最好。
受限环境: 尝试 Netcat FIFO 或编码后的载荷。

终端操作

基本操作

进入终端后，可以直接输入命令执行。支持：

  • 命令历史（上下箭头）
  • 输出滚动查看
  • 实时输出显示

快捷命令

终端页面提供常用命令快速按钮：

whoami id hostname uname -a pwd ls -la ps aux netstat ifconfig

升级为交互式Shell

默认获得的是非交互式Shell，可以升级为完整的TTY：

# Python方法
python -c 'import pty;pty.spawn("/bin/bash")'

# 使用script
script /dev/null -c bash

# 然后按 Ctrl+Z 暂停，执行：
stty raw -echo; fg
export TERM=xterm

常用操作命令

命令 说明
whoami 查看当前用户
id 查看用户权限
uname -a 系统信息
cat /etc/passwd 用户列表
sudo -l 查看sudo权限
find / -perm -4000 查找SUID文件
crontab -l 查看计划任务

防火墙配置

端口检测

在"端口管理"页面可以检测端口是否可用：

  1. 输入端口号
  2. 点击"检测端口"
  3. 查看端口是否被占用

开放防火墙端口

系统支持自动配置防火墙规则：

Windows

# 使用netsh命令
netsh advfirewall firewall add rule name="ShellPort_4444" dir=in action=allow protocol=tcp localport=4444

Linux

# 使用iptables
iptables -I INPUT -p tcp --dport 4444 -j ACCEPT

# 或使用ufw
ufw allow 4444/tcp

# 或使用firewalld
firewall-cmd --add-port=4444/tcp --permanent
firewall-cmd --reload
注意：防火墙操作需要管理员/root权限。

云服务器配置

如果使用云服务器，还需要在云平台控制台的安全组中开放端口：

  • 阿里云：安全组规则
  • 腾讯云：安全组
  • AWS：Security Groups
  • Azure：Network Security Groups

常见问题

Q: 连接不上监听器？

可能原因：

  • 防火墙未开放端口 - 使用端口管理功能开放端口
  • 目标机器无法访问您的IP - 检查网络连通性
  • 载荷中的IP/端口错误 - 确保使用正确的IP和端口
  • 目标机器没有执行载荷的权限 - 尝试提权

Q: 连接后没有Shell提示符？

这是正常现象。您可以直接输入命令执行。如果需要完整的交互式Shell，请使用Python升级：

python -c 'import pty;pty.spawn("/bin/bash")'

Q: 如何获取本机IP？

# Windows
ipconfig

# Linux/macOS
ifconfig
# 或
ip addr show

Q: 支持IPv6吗？

目前仅支持IPv4。

Q: 可以穿透NAT/内网吗？

需要端口转发或使用VPS。监听器需要在公网可访问的地址上运行。

Q: 连接经常断开？

可能原因：

  • 网络不稳定
  • 目标机器有连接超时限制
  • 防火墙/杀毒软件干扰

可以在设置中开启自动重连功能。

Q: 如何在后台运行？

# Linux - 使用nohup
nohup python app.py &

# Linux - 使用screen
screen -S shellmgr
python app.py
# Ctrl+A+D 分离

# Windows - 使用任务计划程序或注册为服务
